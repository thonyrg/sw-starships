Tech Specs

This application has been developed using in Angular 7, Twitter's Bootstrap 4 for RWD (Responsive web design) using mobile first approach and Karma/Jasmine for unit testing

How to run it locally

  1. Install Angular-cli globally on your computer using npm install -g @angular/cli (you may need sudo permissions if you're on Mac or Linux
  2. Clone the git repo at: https://github.com/thonyrg/sw-starships.git
  3. Go to the application folder and run npm install to install dependencies
  4. Use the command ng serve -o to run and to open the application automatically
  5. To run unit tests, use the command ng test

This application consists in the following parts

Components
  • Header Component (/src/shared/components/header/ folder)
    • Component created to show the application header
  • Menu Component (/src/shared/components/menu/ folder)
    • Component created to show the application menu
  • Home Component (/src/home/ folder)
    • Component created to be the landing page where there are instructions on how to use the application
  • Calculate Component (/src/calculate/ folder)
    • Component created to show the starships form and results
    • This component receives the results from the Starship Form component and send them to Starships component
  • Starships Form Component (/src/calculate/starships-form folder)
    • Component created to show the starships form where the user can choose one or all starships and type in the distance wanted to be calculated
    • Once the calculation is done, this component is responsible for emitting the results to the Calculate component
  • Starships Component (/src/calculate/starships folder)
    • Component created to show the list of starships as a result of the distance calculation
    • This component receives the results from the Calculate component and send each item to the Starship component
    • It renders a list of Starship components
  • Starship Component (/src/calculate/starships/starship folder)
    • Component created to show a card containing starship information
    • This component receives a starship information from the Starships component
Services
This application has only one service (/src/app/shared/services/sw-starships.service.ts) with the following methods:
  • getStarships
    • Parameters:
      • specificEndpoint: string with default value of '' (empty string)
      • If a specific endpoint is not provided, it uses the starships endpoint to fetch the initial data
    • Responsible for doing the HTTP request to fetch the list of starships from the swapi (https://swapi.co/starships/)
    • This method returns an observable
  • getHoursFromPeriod
    • Parameters:
      • period: string with default value of '' (empty string)
      • If a specific endpoint is not provided, it uses the starships endpoint to fetch the initial data
    • Responsible for converting periods of time (e.g: 2 years or 4 weeks) in hours
    • This method returns a number
  • getNumberOfStopsForResupply
    • Parameters:
      • distance: number
      • consumablesInHours: number
      • mglt: string as the mega light value may be 'unknown'
      • All parameters are required
    • Responsible returning the number of stops for resupply a starship will need
    • This method returns a string
